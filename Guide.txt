### The Root Cause
In your `auth-context.tsx`, the `getUsers` function fetches data from Supabase and saves it to IndexedDB. However, the filtering logic relies on `slugifyOrg`. If the `organizationName` stored in the user's profile (e.g., "ACRUX IT SOLUTIONS") doesn't match the slugified version expected by the filter (e.g., "acrux-it-solutions"), or if the `state.user.organizationName` is not yet updated when the page loads, the list will appear empty.

Additionally, the **System Setup Button** logic in your main `page.tsx` (the auth page) correctly checks for admins, but the **User Management Page** needs to be more robust in ensuring it only shows users from the *current* admin's organization.

#### 1. Update `lib/auth-context.tsx`
Replace the existing `getUsers` function with this more robust version that ensures consistency between local and remote data:

```tsx
// Inside lib/auth-context.tsx

const getUsers = async () => {
  const db = await getDB();
  // Ensure we have a valid organization name to filter by
  const currentOrgName = state.user?.organizationName;
  
  if (!currentOrgName) {
    console.warn("No organization name found for current user");
    return [];
  }

  const currentOrgSlug = slugifyOrg(currentOrgName);

  // 1. Try to fetch from Supabase if online to get the latest data
  if (navigator.onLine) {
    try {
      const { data, error } = await supabase
        .from('user_profiles')
        .select('*')
        .eq('organizationName', currentOrgSlug); // Filter by slugified org name
      
      if (!error && data) {
        // Update IndexedDB with the latest remote data
        for (const profile of data) {
          await db.put('user_profiles', profile);
        }
        // Return the fresh remote data
        return data;
      }
    } catch (err) {
      console.error("Failed to fetch users from Supabase:", err);
      // Fall through to local DB if remote fails
    }
  }

  // 2. Fallback to IndexedDB (Offline-First)
  const profiles = await db.getAll('user_profiles');
  
  // Filter locally to ensure only users from the current organization are returned
  return profiles.filter(p => {
    const org = p.organizationName || p.orgId;
    // Check both direct match and slugified match for safety
    return org === currentOrgName || slugifyOrg(org) === currentOrgSlug;
  });
};
```
