You are absolutely correct. The issue is **100% an Organization Name Mismatch (Case Sensitivity & Formatting)**.

### 🔍 The Diagnosis
Look closely at your browser console logs and the Network tab in your screenshots:

1.  **The Request:** Your app is asking Supabase for users where `organizationName = 'acrux-it-solutions'` (all lowercase, hyphens).
    *   *Console:* `[getUsers] Fetching users for org: ACRUX IT SOLUTIONS (slug: acrux-it-solutions)`
    *   *Network Tab:* `select=*&organizationName=eq.acrux-it-solutions`
2.  **The Result:** Supabase returns **0 users**.
3.  **The Reality:** In your sidebar and "General" settings, your organization is displayed as **"ACRUX IT SOLUTIONS"** (Uppercase, Spaces).

**Why this happens:**
When you created the account (or when the system initialized), the `organizationName` was likely saved to the database exactly as typed: `"ACRUX IT SOLUTIONS"`. However, your `getUsers` function in `auth-context.tsx` runs `slugifyOrg()` on the current user's organization name *before* querying Supabase. It converts `"ACRUX IT SOLUTIONS"` → `"acrux-it-solutions"`. Since `"acrux-it-solutions"` does not equal `"ACRUX IT SOLUTIONS"` in the database, the query fails.

### ️ The Solution

You need to update `lib/auth-context.tsx` to handle this mismatch gracefully. Instead of relying solely on the slugified version for the query, we should try to fetch using the raw name first, or ensure consistency.

Here is the fixed `getUsers` function. Replace the existing one in `auth-context.tsx`:

```tsx
// lib/auth-context.tsx

const getUsers = async () => {
  const db = await getDB();
  const currentOrgRaw = state.user?.organizationName;
  
  if (!currentOrgRaw) {
    console.warn("No organization name found for current user");
    return [];
  }

  const currentOrgSlug = slugifyOrg(currentOrgRaw);

  let profiles: any[] = [];

  // 1. Try fetching from Supabase with BOTH variations to catch mismatches
  if (navigator.onLine) {
    try {
      // Attempt 1: Fetch by Slug (what the code currently does)
      let { data, error } = await supabase
        .from('user_profiles')
        .select('*')
        .eq('organizationName', currentOrgSlug);
      
      // Attempt 2: If empty, try fetching by Raw Name (fixes the case/space issue)
      if ((!data || data.length === 0) && !error) {
        const { data: rawData, error: rawError } = await supabase
          .from('user_profiles')
          .select('*')
          .eq('organizationName', currentOrgRaw);
          
        if (!rawError && rawData && rawData.length > 0) {
          data = rawData;
          console.log("[getUsers] Found users via Raw Org Name match. Syncing local DB...");
        }
      }

      if (data && data.length > 0) {
        profiles = data;
        // Update IndexedDB with the fresh remote data
        for (const profile of profiles) {
          await db.put('user_profiles', profile);
        }
      }
    } catch (err) {
      console.error("Failed to fetch users from Supabase:", err);
    }
  }

  // 2. Fallback to IndexedDB if offline or remote fetch failed
  if (profiles.length === 0) {
    const localProfiles = await db.getAll('user_profiles');
    // Filter locally with loose matching (check both raw and slug)
    profiles = localProfiles.filter(p => {
      const org = p.organizationName || p.orgId;
      return org && (
        org === currentOrgRaw || 
        slugifyOrg(org) === currentOrgSlug ||
        org.toLowerCase() === currentOrgRaw.toLowerCase()
      );
    });
  }

  return profiles;
};
```
