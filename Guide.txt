To ensure your application never mixes uppercase and lowercase organization names, causing users to "disappear" from the User Management page, you must enforce **strict slugification at the source**.

The root cause of your issue is that `organizationName` is being stored inconsistently: sometimes as `"ACRUX IT SOLUTIONS"` (raw input) and sometimes as `"acrux-it-solutions"` (slugified). When Supabase queries for one format but the data is in the other, it returns 0 results.

Here is the definitive fix to make organization handling uniform across your entire app.

### 1. Enforce Uniformity in `auth-context.tsx`
Your `getUsers`, `login`, and `register` functions must **always** use the slugified version when querying or storing data. Never pass raw organization strings to Supabase.

```typescript
// In auth-context.tsx, update these critical functions:

const getUsers = async () => {
  if (!user) return [];
  
  // ALWAYS use slugified org for queries
  const orgSlug = slugifyOrg(user.organizationName);
  
  const { data, error } = await supabase
    .from('user_profiles')
    .select('*')
    .eq('organizationName', orgSlug); // ← Critical: Query by slug
  
  if (error) {
    console.error('Failed to fetch users:', error);
    return [];
  }
  
  return data.map(p => ({
    id: p.id,
    staffId: p.staffId,
    name: p.name,
    role: p.role,
    isAdmin: p.isAdmin,
    organizationName: p.organizationName, // Store as-is from DB (should be slug)
    department: p.department,
  }));
};

const login = async (staffId: string, password: string, org: string) => {
  try {
    const email = getDummyEmail(staffId);
    const orgSlug = slugifyOrg(org); // ← Normalize input immediately
    
    const { data, error } = await supabase.auth.signInWithPassword({ email, password });
    if (error || !data.session) {
      return { success: false, error: error?.message || 'Invalid credentials' };
    }
    
    const meta = data.user.user_metadata || {};
    
    // Verify organization matches using SLUG comparison
    if (slugifyOrg(meta.organizationName) !== orgSlug) {
      await supabase.auth.signOut();
      return { success: false, error: 'Organization mismatch' };
    }
    
    // ... rest of login logic remains the same
  } catch (err: any) {
    return { success: false, error: err.message };
  }
};

const register = async (name: string, staffId: string, password: string, role: string, org: string, dept: string) => {

      .from('user_profiles')
      .select('id')

  } catch (err: any) {
    return { success: false, error: err.message };
  }
};
```

### 4. Database-Level Enforcement (SQL)
Add a check constraint to prevent non-slugified values from ever entering the database:

```sql
-- Run in Supabase SQL Editor
ALTER TABLE public.user_profiles 
  DROP CONSTRAINT IF EXISTS user_profiles_org_format_check;

ALTER TABLE public.user_profiles 
  ADD CONSTRAINT user_profiles_org_format_check 
  CHECK ("organizationName" ~ '^[a-z0-9-]+$');
```

This regex ensures only lowercase letters, numbers, and hyphens are allowed — exactly what `slugifyOrg()` produces.
